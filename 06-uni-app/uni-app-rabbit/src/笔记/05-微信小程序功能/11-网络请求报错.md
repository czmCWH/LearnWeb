# 网络请求报错

## cronet_error_code:-207 ... errno: 600001

使用 `wx.request` <https://developers.weixin.qq.com/miniprogram/dev/api/network/request/wx.request.html> 发送网络请求，在 iOS 14.5 版本报错如下：
```
{errMsg: "request:fail errcode:-207 cronet_error_code:-207 error_msg:net::ERR_CERT_INVALID", errno: 600001}errMsg: "request:fail errcode:-207 cronet_error_code:-207 error_msg:net::ERR_CERT_INVALID"errno: 600001__proto__: Object
```
在 iOS18、26版本、android、浏览器都是正常的。咨询微信客服说技术客服他们不负责，社区链接：<https://developers.weixin.qq.com/community/develop/mixflow>

使用 <https://myssl.com/> 检测  MySSL 证书提示：

```
降级原因：
1.因为使用RC4密码套件，降级为B
```

### 解决

#### 方法1，换证书，更换证书颁发机构。
```
acme.sh --server https://acme-v02.api.letsencrypt.org/directory   生成证书的时候指定为  Let’s Encrypt
ZeroSSL 换  Let’s Encrypt
```

#### 方法2，ios对证书要求很高，ios一顿设置，挺麻烦的。不推荐！
